# Priority

A personal productivity app built with Flutter + Firebase. Three views — Notes, Eisenhower Matrix, Goals — with real-time sync and Google Sign-In.

**Firebase project:** `priority-d553f` | **Web URL:** https://priority-d553f.web.app

> Full command reference: [COMMANDS.md](COMMANDS.md) | Known issues & roadmap: [TODO.md](TODO.md)

## Features

### Notes (Google Keep-style)
- Masonry grid layout with pinned notes always on top
- Search across all notes in real time
- Checklists with items that grey out and sink to the bottom when checked
- Color coding with 10 note colors
- Offline support — all changes sync when you reconnect

### Eisenhower Matrix
- 2×2 grid: DO NOW · SCHEDULE · DELEGATE · ELIMINATE
- Deadlines with overdue highlighting
- Subtasks per task
- Responsive layout — grid on tablet/web, list on mobile
- Completed tasks grey out and move to the bottom of their quadrant

### Goals
- Short-term and long-term goal tracking
- Milestones with target dates
- Progress bar auto-computed from completed milestones
- Expandable milestone list per goal

### Cross-platform
- Android (primary — Google Play Store)
- Web (Firebase Hosting) — access from any browser
- iOS (planned)

### Security
- Google Sign-In via Firebase Authentication
- TOTP 2-factor authentication — code is ready, requires Firebase Blaze plan upgrade to activate (see [TODO.md](TODO.md))
- Firestore security rules: users can only access their own data
- All data encrypted in transit (HTTPS) and at rest (Firebase)

### Sync
- Real-time sync across all devices via Firestore listeners
- Full offline support — work without internet, changes sync automatically

## Getting Started (fresh machine setup)

### Prerequisites

1. [Flutter SDK](https://flutter.dev/docs/get-started/install) (stable channel, ≥ 3.3.0)
2. Android Studio or VS Code with the Flutter extension
3. [Firebase CLI](https://firebase.google.com/docs/cli): `npm install -g firebase-tools`
4. [flutterfire CLI](https://firebase.flutter.dev/docs/cli): `dart pub global activate flutterfire_cli`

### Firebase Setup

1. Create a project at [Firebase Console](https://console.firebase.google.com)
2. Enable **Authentication** → Sign-in providers:
   - Google Sign-In
   - Multi-factor Authentication → TOTP
3. Create a **Firestore Database** (choose production mode)
4. Register an **Android app** (package: `com.ssharma.priority`)
5. Register a **Web app**

### Installation

```bash
# Clone / open the project
cd priority

# Connect to your Firebase project
flutterfire configure
# This generates lib/firebase_options.dart and downloads google-services.json

# Install dependencies
flutter pub get

# Generate freezed/json_serializable code
dart run build_runner build --delete-conflicting-outputs

# Deploy Firestore security rules
firebase deploy --only firestore:rules,firestore:indexes
```

### Running

```bash
# Android emulator or device
flutter run

# Web browser
flutter run -d chrome
```

## Deployment

### Web (Firebase Hosting)

```bash
flutter build web --release --web-renderer canvaskit
firebase deploy --only hosting
```

Your app will be live at `https://<project-id>.web.app`

### Android (Google Play Store)

```bash
# 1. Generate signing key (one time)
keytool -genkey -v -keystore upload-keystore.jks -alias upload -keyalg RSA -keysize 2048 -validity 10000

# 2. Create android/key.properties (DO NOT COMMIT)
# storePassword=...
# keyPassword=...
# keyAlias=upload
# storeFile=../upload-keystore.jks

# 3. Build release bundle
flutter build appbundle --release

# 4. Upload to Google Play Console
```

## Security Model

**Two layers of protection:**

1. **Firebase Auth + TOTP 2FA**
   Every user signs in with their Google account. After the first sign-in, they must enroll a TOTP authenticator app (e.g. Google Authenticator). On all subsequent sign-ins, a 6-digit one-time code is required.

2. **Firestore Security Rules**
   All data is stored under `/users/{uid}/`. Server-side security rules enforce that `request.auth.uid == uid` on every read and write. No user can access another user's data even if they craft direct API calls.
